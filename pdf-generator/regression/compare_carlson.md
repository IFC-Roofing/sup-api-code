## ROOT CAUSE ANALYSIS

### CRITICAL ISSUES

1. **DOUBLE O&P ON BID ITEMS** — This is the biggest finding. Fence: Sup retail $1,846.80 = Human's total (retail $1,539 + O&P $307.80). Pergola: same pattern. Sup is taking the number that ALREADY includes O&P and treating it as the retail, then calc_line_item adds O&P again. This inflates bids by 20%. On Carlson: fence $369 + pergola $299 = ~$668 of phantom O&P.

2. **GUTTER AS BID ITEM** — Violates the "always Xactimate" rule. Pipeline should have caught this — gutters are in XACTIMATE_TRADES. Either the bid wasn't tagged correctly or the filter didn't work.

3. **MISSING HIGH ROOF CHARGES** — $2,000 left out. EV shows 2+ stories. Opus prompt says "ALWAYS add high roof if 2+ stories." Post-processor doesn't enforce this.

4. **FELT WITH WASTE** — Sup used 55.33 SQ (suggested with waste). Felt should be measured SQ (49 SQ). The _fix_steep_waste post-processor has felt correction code but it seems like it calculated wrong here — added 13% waste to felt instead of keeping it at tear-off qty.

5. **STEP FLASHING F9 SAYS HAIL DAMAGE** — Direct violation of domino rule. F9 says "per hail damage" instead of "disturbed during tear-off, cannot be reused."

6. **WRONG SUB NAME** — HVAC bid shows "Services" instead of "Polar Pros." Pipeline extracted the wrong name from the bid PDF.

### MODERATE ISSUES

7. **R&R vs install-only on tear-off** — Same as Baswell. Tear-off has no remove component.

8. **Apron flashing wrong description** — "Drip edge/gutter apron" instead of "R&R Counterflashing - Apron flashing." The _fix_descriptions post-processor should catch this but the pattern "drip edge/gutter apron" doesn't match its correction map.

9. **Human excluded starter/hip/ridge/valley** — This is interesting. Sup added them (which is correct per the pipeline rules), but human chose not to. Either strategic choice or human missed them. If strategic, we need to understand why.
